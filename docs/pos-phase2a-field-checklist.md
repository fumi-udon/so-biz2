# POS Phase 2-A — 実装後フィールドチェックリスト

本番・ステージングでの手動確認用。項目は **主に卓ホスト（`table-action-host`）と posDraft 残像**を対象とする。

---

## A. Console / 残像フロー（5 ステップ）

1. **卓 A をタップ** — キャッシュ Miss ならパルススケルトン、Hit なら残像パネル＋「🔄」付近の同期バッジ。
2. **`pos-action-host-authoritative` 受信後** — `lineSurfaceMode` が実質 `live` 相当となり、Livewire の行一覧が表示され、バッジが消えること。
3. **別卓 B をタップ** — 同様に Miss ならスケルトン、Hit なら残像。
4. **卓 A に戻る** — LRU に残っていれば **Hit** で残像が即時表示＋バッジ。
5. **再度 authoritative 受信** — `live` に戻りバッジ消滅。

（開発者向け: `window.addEventListener('pos-action-host-authoritative', e => console.log(e.detail))` で detail を追跡可能。）

---

## B. HIT_STALE 中の操作（`lineSurfaceMode === 'afterimage'`）

| 操作 | 期待 |
|------|------|
| **Ajouter** | **押下可能**（`afterimage` を disabled 式に含めない） |
| **Recu（bulk）** | disabled |
| **行削除（×）** | disabled |
| **Addition / Clôture** | disabled |
| **割引起動** | 本画面に専用ボタンが無い場合はスキップ。別導線がある場合は同方針で確認 |

---

## C. F5（ページ離脱）警告

| 状況 | 期待 |
|------|------|
| 何も入力せず卓のみ | 離脱警告 **なし**（またはアプリ既定どおり問題なし） |
| **商品追加モーダルが開いている** | 離脱・再読込時に **警告が出る** |
| **モーダル内で入力したあと** | 同上で **警告が出る** |

ブラウザの「サイトを再読み込みしますか」相当の挙動を確認する。

## E. レスポンシブ

| 幅 | 確認 |

| **約 768px** | レイアウト切替（例: 2 カラム POS）で残像／live 行が重ならない |

## F. ライト / ダーク

- テーマ切替で残像パネル・バッジ・スケルトン・disabled ボタンのコントラストが読めること。

---

## G. HIT_STALE 中の Ajouter — 主観評価（2-B 持ち越し用）

- **モーダル背面が残像のまま**表示されることによる違和感の有無を現場で評価する。
- **違和感がある**場合は Phase **2-B** で UX（背面を live に寄せる、半透明オーバーレイ等）を再判断する。現時点では **Ajouter 許可**を維持する仕様。
